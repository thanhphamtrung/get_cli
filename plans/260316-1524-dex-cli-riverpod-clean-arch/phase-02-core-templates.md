## Implementation Steps

### Step 1: Create Sample Template Files
Create all 19 template files in `lib/samples/impl/riverpod_clean/`. Each extends `Sample` base class.

Key template patterns:
- All providers: `part '{name}.g.dart'` + `@riverpod` annotation
- All models: `part '{name}.freezed.dart'` + `part '{name}.g.dart'` + `@freezed`
- All entities: `part '{name}.freezed.dart'` + `@freezed` (no JSON)
- Repositories: `Either<Failure, T>` return types (fpdart)

### Step 2: Create Init Command
Create `init_riverpod_clean.dart`:
1. Generate `main.dart` (ProviderScope + MaterialApp.router)
2. Install all packages via pubspec update
3. Create `lib/core/` directory with error, network, router, config, usecases
4. Create initial `home` feature module (all layers)
5. Create `lib/shared/` directories
6. Create `.env` and `.env.example`
7. Update `.gitignore` (add .env, *.g.dart, *.freezed.dart)
8. Create `build.yaml` for build_runner config

### Step 3: Update Structure Definitions
In `lib/core/structure.dart`, add feature-first paths:
```dart
// Feature paths
static String featurePath(String name) => 'lib/features/$name';
static String dataPath(String name) => 'lib/features/$name/data';
static String domainPath(String name) => 'lib/features/$name/domain';
static String presentationPath(String name) => 'lib/features/$name/presentation';
static String corePath = 'lib/core';
static String sharedPath = 'lib/shared';
```

### Step 4: Update Init Command Router
In `init.dart`, replace GetX/Arktekko options with single Riverpod Clean Arch init.

### Step 5: Create pubspec Template
Template for generated project's `pubspec.yaml` with all agreed packages:

## Risk Assessment
- **Package version pinning**: Versions may drift. Pin ranges (^x.y.0) not exact versions.
- **build_runner conflicts**: freezed + riverpod_generator + json_serializable all use build_runner — need proper `build.yaml` config.
- **Template string escaping**: Dart templates with `$` variables need careful escaping in hardcoded strings.
